# 北京邮电大学 2026 年电子设计竞赛小车赛道题目设计报告

项目名称：巡迹作业一体平台

队伍/成员：朱锦松、郑洢陈、杨照基

日期：2026 年 7 月 9 日

## 摘要

本项目面向“巡迹作业一体平台”题目，设计并制作一套以 TI MSPM0G3507 为主控制器的轮式小车系统。系统由差速轮式底盘、非视觉巡迹模块、编码器/IMU 辅助运动控制模块、二维云台/作业执行模块（含 K230 视觉辅助定位）、多模块独立供电系统及人机交互模块组成。小车通过 Yahboom 8 路灰度传感器识别黑色轨迹，结合编码器里程计和 MPU6050 姿态信息完成 A→B→C→D→A 自动运行，在 B 点和最终 A 点通过 MSPM0-K230 GPIO 握手协议触发视觉瞄准与激光作业。

本项目的开发历程体现了典型的工程实践特点：团队最初以"2+1"（自动巡迹 + 定点作业 + 同步绘制）为保底目标，在后期根据比赛要求通宵扩展至"3+2"（增加巡迹到靶位联动和四圈连续运行），最终除因缺少无刷云台硬件而无法实现的动态跟随外，其余功能全部完成。开发期间经历了搬家导致硬件损坏、陀螺仪到货延迟、第一次参赛硬件经验不足等现实困难，团队在每一阶段都通过模块化验证和回归测试保持了底盘控制基线的稳定。

## 1. 题目要求分析

题目要求设计一套“巡迹作业一体平台”，装置应由轮式小车底盘、巡迹控制模块和云台或机械臂等作业模块构成。小车需要在指定黑线场地内自动巡迹行驶，并在指定位置完成对目标靶的指向或标记动作。

### 1.1 场地与目标靶

比赛场地尺寸不小于 220 cm x 120 cm，表面为白色哑光平面材料。轨迹由两段直线或虚线和两段对称半圆弧组成，线宽为 1.8 cm ± 0.2 cm，并设置 A、B、C、D 四个关键点。目标靶设置在 AB 线段外侧 50 cm 处，靶面与 AB 平行，高度不大于 50 cm，靶面为 A4 记录材料并带有靶心和同心圆辅助判定区域。

### 1.2 约束条件

作品外形尺寸不大于 25 cm x 15 cm x 25 cm；底盘必须为轮式结构，不得使用履带或麦克纳姆轮；主控制器必须采用 TI 芯片；小车路径识别和目标作业不得依赖摄像头；行驶过程中不得人为干预，也不得使用外部导航设备。云台或机械臂作业模块需要具备独立电源控制，并满足基本安全要求。

### 1.3 基本要求

1. 自动巡迹：小车从 A 点出发，沿轨迹行驶一圈并回到 A 点停车，停车时给出声光提示，完成时间不大于 30 s。
2. 定点瞄准：小车放置在轨迹指定位置并启动后，应在 5 s 内自动调整云台或机械臂姿态，使执行器末端对准靶心并完成一次指定动作。
3. 巡迹到靶位联动：小车从 A 点出发，先巡迹至 B 点停车并完成一次对靶作业，然后继续经 C、D 回到 A 点，每经过关键点均给出声光提示，一圈总时间不大于 40 s。
4. 工程训练项：底盘、云台/机械臂和人机交互模块应支持独立上电、单独测试，并能通过按键或串口设置运行模式和参数。

### 1.4 发挥要求

发挥部分包括动态瞄准、同步绘制、四圈连续运行和工程完善度。动态瞄准要求小车沿轨迹行驶时云台或机械臂保持对目标靶的持续跟随；同步绘制要求静止时在靶面绘制清晰图案；四圈连续运行要求系统在完成一次完整瞄准任务的前提下连续运行 4 圈；工程完善度关注走线、结构强度、维护便捷性、供电隔离、急停逻辑和故障恢复能力。

## 2. 总体方案

本设计采用“非视觉巡迹 + 编码器里程计 + IMU 航向保持 + 二维云台作业”的总体路线。系统以 MSPM0G3507 为主控制器，底盘采用差速轮式结构，巡迹使用 Yahboom 8 路灰度传感器，驱动使用 TB6612FNG 双路直流电机驱动模块，作业机构采用二维 PWM 舵机云台并预留 405 nm 激光或 5 V 聚光 LED 的安全开关控制接口。

总体功能链路如下：

1. 灰度传感器采集黑线位置，输出 8 位线位置状态。
2. 主控根据线位置误差运行 PD 巡线控制，生成左右轮差速 PWM。
3. 编码器统计左右轮累计脉冲，用于白区直行、关键点窗口判断和一圈停止判断。
4. MPU6050 提供短时间航向保持，使 AB、CD 等无黑线白区直行段不依赖摄像头。
5. 自动状态机按 AB_STRAIGHT、B_GUARD、BC_LINE、C_ALIGN、CD_STRAIGHT、D_GUARD、DA_LINE 的顺序完成 ABCDA 联动。
6. 蜂鸣器、外部 LED、板载 LED、按键和 UART 串口提供调试、启动、故障与完成提示。

该路线符合题目对 TI 主控、轮式底盘、非视觉路径识别和模块独立调试的约束，也为后续云台定点作业、动态跟随和同步绘制保留了可扩展接口。

## 3. 硬件设计

### 3.1 主控制器

主控制器选用 MSPM0G3507 核心开发板。该芯片属于 TI MSPM0 系列，具备 GPIO、定时器 PWM、UART、I2C 和中断输入等资源，可满足电机驱动、灰度传感器读取、编码器计数、IMU 通信、蜂鸣器/LED 控制和按键输入需求。

主控板在调试阶段通过 USB 供电并使用 J-Link SWD 下载程序。项目调试中已经完成 Keil 工程构建、J-Link 烧录、板载 PB22 LED 健康闪烁和 CH340 串口输出验证。

### 3.2 底盘与驱动

底盘采用轮式差速结构，当前优先使用 2WD 方案进行稳定巡线和关键点控制。电机为 12 V 520 编码器电机，左右轮分别由 TB6612FNG 双通道驱动模块控制。早期调试中已经确认两路电机均可正反转和制动，且编码器在前进时可输出同向递增的逻辑计数。

现阶段已验证的电机和编码器接口如下：

| 功能 | 接口/引脚 |
| --- | --- |
| 左/右电机 PWM | PA8、PA9 |
| 电机方向控制 | PA16、PA21、PA22、PA23 |
| 电机待机控制 | PB20 |
| 左编码器 A/B | PB17、PB18 |
| 右编码器 A/B | PB24、PB25 |

控制策略上，电机输出支持空转停止和 TB6612 短刹车两种方式。在关键点停车、故障保护和最终完成时优先使用短刹车，以减少惯性滑行对停车位置的影响。

### 3.3 巡迹传感器

巡迹模块采用 Yahboom YB-MYX05-V1.0 8 路灰度传感器。该传感器为 3 位地址选择加 1 路 OUT 输出的复用结构，通过 AD0、AD1、AD2 选择通道，并从 OUT 读取当前通道的黑白状态。实测结果为：白色或亮处输出 0，黑色或暗处输出 1，OUT 高电平约为 3.3 V，可直接接入 MSPM0G3507 GPIO。

已验证接口如下：

| 传感器信号 | MSPM0G3507 引脚 |
| --- | --- |
| AD0 | PA26 |
| AD1 | PA27 |
| AD2 | PA24 |
| OUT | PA25 |

固件将 8 个通道采样为 8 位线状态，并根据黑色通道数量和位置计算加权误差。中心通道检测到黑线时误差接近 0；线偏右时误差为正，控制器提高左轮速度并降低右轮速度；线偏左时误差为负，控制器降低左轮速度并提高右轮速度。

### 3.4 IMU 与白区直行

由于题目轨迹包含白色直线/虚线段，单纯依赖灰度传感器会在无黑线区域失去路径反馈。本设计增加 MPU6050 / GY-521 模块用于短时间航向保持。模块通过 I2C 连接主控：

| MPU6050 信号 | MSPM0G3507 引脚 |
| --- | --- |
| VCC | 3V3 |
| GND | GND |
| SCL | PB2 |
| SDA | PB3 |
| AD0 | 悬空或接 GND |

启动自动模式前，固件先执行静止状态下的陀螺仪 Z 轴零偏校准。白区直行时，控制器结合左右轮编码器差值和 yaw 偏差进行修正，使小车在没有黑线反馈的 AB、CD 段仍能保持直行。

### 3.5 云台与执行模块

作业模块规划为二维 PWM 舵机云台，末端可安装激光指示器、聚光 LED 或轻量标记结构。云台不从主控板取电，而使用独立 5 V 至 6 V 高电流电源；舵机信号地与主控共地，舵机电源正极与主控电源隔离。激光或照明模块不得由 GPIO 直接供电，需通过 MOSFET 或负载开关控制，并增加默认关闭、物理断电和靶面吸收等安全措施。

该设计满足题目中“云台/机械臂模块应具备独立电源控制”的要求。当前项目已经完成云台、电源和激光安全接入方案设计，执行机构的定点瞄准和绘制功能待在底盘稳定后进行联调。

### 3.6 人机交互

系统提供三类人机交互：

1. UART 串口：用于查看传感器、电机、编码器、IMU 和状态机遥测，也可调整模式和参数。
2. B21 按键：用于 3 s 倒计时启动、取消启动、触发静止 IMU 校准并进入自动 A 模式。
3. 声光提示：PA2 有源低电平蜂鸣器、PA14 外部 LED 和 PB22 板载 LED 用于启动、完成、失败和健康状态提示。

当前声光反馈规则为：开机完成给出一次短提示；自动运行开始给出一次短提示；IMU 校准成功给出两次短提示；自动运行完成给出较长提示并快速闪烁；自动运行失败给出三次短提示并快速闪烁。

## 4. 软件设计

### 4.1 软件结构

固件基于 TI DriverLib/SysConfig 和 Keil 工程开发，主要功能包括：

1. GPIO、PWM、UART、I2C 和中断初始化。
2. 灰度传感器复用采样与线位置误差计算。
3. 左右轮 PWM 输出、方向控制和短刹车控制。
4. 左右编码器计数和逻辑方向修正。
5. MPU6050 初始化、gyroZ 零偏校准和 yaw 积分。
6. 巡线 PD 控制、白区直行控制和 ABCD 自动状态机。
7. 串口命令解析、遥测输出、按键消抖和声光提示。

当前主力自动运行工程为 `firmware/abcd_four_lap_finish_aim`，它基于已验证的 ABCD 底盘运动基线，增加了四圈连续运行和最终 A 点 K230 视觉瞄准功能。固件通过 MSPM0 的 PB11 和 PB14 引脚与 K230 视觉模块进行 GPIO 握手通信，实现 B 点定点作业和最终 A 点瞄准的自动触发与完成确认。

### 4.2 巡线控制

巡线模式以 20 ms 为控制周期。传感器读取 8 位状态后，先判断 `OK`、`LOST`、`FULL` 三类情况：正常黑线为 `OK`，全白为 `LOST`，全黑为 `FULL`。正常情况下计算加权误差，并使用 PD 控制生成转向修正量：

```text
correction = Kp * error + Kd * (error - last_error)
left_pwm  = base + correction
right_pwm = base - correction
```

当前低速巡线基准参数为：

| 参数 | 当前值 |
| --- | ---: |
| 控制周期 | 20 ms |
| base | 180 / 1000 |
| Kp | 55 |
| Ki | 0 |
| Kd | 35 |
| 转向限幅 | 240 / 1000 |

在 Step 6 地面测试中，小车已经能够通过 `g` 命令进入巡线模式，并在地面黑线段上完成低速稳定跟踪。该模式也是后续 BC、DA 黑线圆弧控制的基础。

### 4.3 白区直行控制

白区直行模式用于 AB 和 CD 段。由于传感器在白区读数为全白，不能直接提供横向误差，因此采用编码器平衡和 IMU 航向保持共同修正：

1. 编码器速度项：比较最近一个控制周期左右轮增量，补偿瞬时速度差。
2. 编码器位置项：比较从本段起点开始的左右累计增量，补偿长期偏航。
3. IMU 航向项：比较当前 yaw 与目标 yaw，补偿小车相对目标方向的角度偏差。

AB 白区直行目标 yaw 为 0 deg；CD 白区直行使用现场调试得到的固定目标 `-153.00 deg`，避免直接将 C 点出弧瞬间的 yaw 作为唯一方向基准。

项目中通过 `F` 模式对 A-B 白区进行了单独测量，得到黑线入口里程约为 1558 至 1588 tick，和题目中 AB 约 100 cm 的参考长度相符。当前使用 `gapBias=15` 作为白区直行基准偏置。

### 4.4 ABCD 自动状态机与四圈循环

自动联动模式通过 `A` 命令（单圈）、`Q` 命令（四圈+最终瞄准）或 B21 按键启动。启动前必须完成 MPU6050 静止校准（`I` 命令），校准成功后进入 ABCD 状态机。状态流程如下：

```text
AB_STRAIGHT -> B_GUARD -> BC_LINE -> C_ALIGN -> CD_STRAIGHT
            -> D_GUARD -> DA_LINE
```

各状态功能如下：

| 状态 | 功能 |
| --- | --- |
| AB_STRAIGHT | 等待全白起点，使用白区直行控制（编码器平衡 + IMU 航向保持，目标 0°）向 B 前进，里程窗口 1200~1900 tick 内寻找黑线入口 |
| B_GUARD | 进入 B 点附近短保护区（15~45 tick），避免边缘黑线直接造成过大 PID 修正 |
| BC_LINE | 使用已验证的黑线巡迹 PD 控制跟踪 B→C 圆弧，出口通过里程（1500~2600 tick）+ 连续丢线确认（3 个控制周期）判断 |
| C_ALIGN | 离开圆弧后主动对齐 CD 固定航向目标（-150°），编码器平衡关闭，纯 IMU 航向修正 |
| CD_STRAIGHT | 使用白区直行控制向 D 前进，航向目标 -150° |
| D_GUARD / D_ACQUIRE | D 点入口保护和中心确认 |
| DA_LINE | 使用黑线巡迹 PD 控制跟踪 D→A 圆弧 |

**四圈循环逻辑**：DA 弧线出口确认后，若已完成圈数 < 目标圈数（Q 模式为 4），进入 `A_NEXT_ALIGN` 状态，小车以 -303° 航向目标完成下一圈 AB 入口对准，随后调用 `mpu6050_straight_reset()` 将 yaw 归零并重新进入 `AB_STRAIGHT`。每圈 yaw 不累积，避免角度溢出。

**最终瞄准**：第 4 圈 DA 出口后进入 `A_ALIGN` 状态，以 -360° 航向目标做短时对准（最多 150 tick），随后无条件进入 `A_FINISH_WORK`，通过 PB11 拉高 START 信号触发 K230 视觉瞄准。MSPM0 等待 PB14 的 DONE 信号稳定 40ms 后确认完成；若 15s 超时则判定失败。

经现场调参确定的四圈关键参数：

| 参数 | 值 | 说明 |
| --- | ---: | --- |
| CD_YAW | -15000（-150°） | CD 直行航向，从 -153° 微调而来，解决了 D 点右侧偏移 |
| NEXT_AB | -30300（-303°） | 圈间 AB 重入航向，-325°偏右、-300°偏左、-303°平衡 |
| AIM | -36000（-360°） | 最终 A 点瞄准航向 |
| CORR_MAX | 40 | IMU 修正上限，对齐状态关闭编码器平衡 |

**MSPM0-K230 握手协议**：MSPM0 与 K230 通过 3.3V GPIO 直连通信：

| MSPM0 引脚 | K230 引脚 | 方向 | 含义 |
| --- | --- | --- | --- |
| PB11 | GPIO03 | MSPM0→K230 | START：高电平触发 K230 开始瞄准 |
| PB14 | GPIO04 | K230→MSPM0 | DONE：高电平表示瞄准+激光射击完成 |
| GND | GND | — | 共地 |

K230 端运行 MicroPython 脚本（`vision/k230_a_finish_aim_handshake.py`），工作流程为：等待 START 高电平 → 灰度 blob 检测黑色靶框 → 二维舵机迭代逼近靶心 → 瞄准稳定 5 帧后激光点亮 3 秒 → 拉高 DONE → 等待 START 低电平后复位。

**同步绘制功能**：K230 端另一脚本实现正方形绘制。瞄准稳定后，舵机按预设正方形路径分段移动（每边 25 步），激光持续点亮形成清晰轨迹。正方形尺寸参数（`DRAW_SIZE_X/Y=0.15`）和整体偏移量（`SQUARE_LEFT_OFFSET=0.45`）可根据靶面位置调整。

### 4.5 串口命令

常用运行和调试命令如下：

| 命令 | 作用 |
| --- | --- |
| `A` / `a` | 启动 ABCD 自动状态机 |
| `F` | 白区直行到黑线的单段测试 |
| `G` | 连续黑线巡迹测试 |
| `g` | 旧的一圈编码器距离停止测试 |
| `I` | 初始化 MPU6050 并进行 gyroZ 零偏校准 |
| `i` | 读取一次 MPU6050 数据 |
| `Y` / `y` | 独立 IMU 辅助直行测试 |
| `b` | 短刹车 |
| `x` | 空转停止并回到待机 |
| `?` | 打印帮助和参数 |

这些测试命令使底盘、传感器、IMU 和自动状态机可以分层验证，符合题目关于模块独立测试和参数可设置的要求。

## 5. 调试与验证

### 5.1 主控与下载验证

项目已经完成 MSPM0G3507 Keil 工程构建、J-Link SWD 下载和串口调试链路验证。早期调试中，J-Link 能够识别 ARM CoreSight SW-DP，Keil 下载输出显示 Erase、Programming 和 Verify 均成功。板载 PB22 LED 健康闪烁和 UART 输出可用于判断程序是否正常运行。

### 5.2 灰度传感器验证

Yahboom YB-MYX05-V1.0 传感器已经完成黑白输出和电平安全验证。测试结论为：白/亮输出 0，黑/暗输出 1，OUT 高电平约 3.3 V，可直接连接 PA25。固件能够读取 8 个通道并输出 `line=0b...` 遥测，后续巡线控制使用该 8 位结果计算误差。

### 5.3 电机与编码器验证

Step 5 中完成左右电机低速正反转、停止和编码器计数验证。电机驱动通过 TB6612 控制，停车时加入短刹车以减少滑行。编码器逻辑方向经过软件修正后，前进时左右编码器均递增，可用于里程计和白区直行控制。

### 5.4 巡线验证

Step 6 已完成低速地面巡线验证。当前基准参数为 `base=180 Kp=55 Ki=0 Kd=35`，方向修正标志为 `mSwap=1 invA=1 invB=1 sensorFlip=1 eSwap=1 eInvA=1 eInvB=1`。测试中通过 `f` 命令确认前进，通过 `x` 命令停车，通过 `g` 命令进入巡线模式。遥测显示误差和左右轮 PWM 调整方向正确，小车能够在地面跟随黑线运动。

### 5.5 白区与 ABCD 联动验证

Step 7 中完成 A-B 白区直行测量，典型结果如下：

| 测试 | 黑线检测里程 |
| --- | ---: |
| 第一次 A-B 白区测试 | 1587 tick |
| 改进编码器平衡后 | 1573 tick |
| 后续重复测试 | 1576 tick、1558 tick |

这些结果与 AB 参考长度约 100 cm 基本一致，说明编码器里程和白区直行控制可用于关键点检测。

后续 `abcd_fresh` 工程对旧自动状态机进行了重新组织，当前自动状态机已在现场测试中完成过完整 ABCDA 环路。最新记录显示，多次 A 模式运行可完成全流程，其中一次偏斜起步导致 D 附近出现 `entry lost black`，分析认为主要是 D 点入口保护逻辑偏严格，而不是 CD 航向目标明显错误。因此当前底盘自动联动已经具备可恢复基线，后续重点是提高 D 点入口鲁棒性和补充更多重复测试数据。

### 5.6 声光提示与启动流程验证

当前固件已加入 PA2 蜂鸣器、PA14 外部 LED 和 PB21 按键。B21 按键支持倒计时启动、再次按键取消、静止 IMU 校准、校准成功后自动进入 A 模式等流程。声光提示已覆盖开机、启动、校准成功、运行完成和运行失败等关键状态。

## 6. 工程完善度设计

### 6.1 模块独立上电

系统按主控、传感器、电机驱动、云台舵机和执行光源分模块供电。调试阶段主控优先 USB 供电，电机由独立电池/电机电源供电，云台使用单独 5 V 至 6 V 高电流电源。所有模块只共享信号地，避免电机和舵机大电流扰动直接影响主控。

### 6.2 安全默认状态

固件要求电机 STBY、激光、照明和舵机输出在复位和故障状态下保持关闭。自动状态机失败、传感器异常、IMU 丢失或人工急停时，电机进入短刹车或待机状态，并通过声光提示故障。

### 6.3 急停与故障恢复

系统支持串口 `b` 短刹车和 `x` 停止命令；实际比赛前应增加物理电机电源开关或急停开关。故障恢复流程为：刹车停车、声光提示、记录串口遥测、检查传感器/IMU/电机状态、重新校准后再启动。

### 6.4 可维护性

硬件上预留 USB 下载口、UART 调试口、模块电源开关、公共地测试点和传感器/电机插接接口。软件上保留 `F`、`G`、`Y` 等独立测试模式，使白区直行、黑线巡迹和 IMU 辅助直行可以分开回归测试，避免调试自动状态机时破坏已验证底层功能。

## 8. 项目 AI 使用说明

本项目在需求拆解、硬件选型、文档整理、固件开发、调试记录和报告撰写过程中使用了 AI agent 工具 Codex。AI 工具不直接替代实物测试，所有涉及电气安全、引脚连接、运动效果和比赛功能的结论均需通过本地资料、代码、编译或实测结果验证。

### 8.1 使用的工具

主要使用工具为 Codex。它被用于读取题目 PDF、整理需求和评分点、检查硬件资料、辅助编写 MSPM0G3507 固件、分析串口日志、记录调试过程、维护项目文档和生成本设计报告。

### 8.2 使用方式

1. 需求分析：将题目 PDF 提取为文本，整理硬约束、基本要求、发挥要求和线上提交材料清单。
2. 硬件规划：根据已购买器件建立硬件清单，区分已知参数、未知参数和需要实测的安全风险。
3. 接线与安全检查：为 MSPM0G3507、Yahboom 灰度传感器、TB6612、电机编码器、MPU6050、舵机和激光/照明模块建立分阶段接线和上电计划。
4. 固件开发：在已有 Keil/SysConfig 工程中实现传感器采样、电机 PWM、编码器计数、巡线 PD、白区直行、IMU 校准、ABCD 自动状态机、声光提示和按键启动逻辑。
5. 调试分析：根据串口日志和现场现象分析 B、C、D 转换失败原因，区分硬件方向、电机控制、传感器识别和状态机逻辑问题。
6. 项目管理：持续更新 roadmap、risk register、bring-up record、AI usage log 和自动调试 handoff，保证长时间调试后仍能恢复上下文。
7. 报告撰写：基于题目要求、现有项目文档和实测记录生成中文设计报告初稿。

### 8.3 对项目的帮助

AI 工具显著提高了需求拆解和调试记录的效率。它帮助团队把题目从自然语言转化为可执行的工程检查项，把硬件风险提前整理为上电顺序和安全默认状态，并在固件调试中保存每次失败的现象、原因和下一步测试顺序。对于 ABCD 自动状态机这类长流程问题，AI 还帮助把已验证的 F/G/Y 单项测试与自动模式隔离，减少对已通过功能的破坏。

### 8.4 局限性与控制措施

AI 工具无法直接观察实物电路，也不能替代万用表测量、示波器验证、现场跑车和安全评估。它可能根据常见模块经验做出不适用于本项目具体器件的假设，例如电机驱动板引脚、电池放电能力、舵机电流、激光供电方式和编码器 CPR 等。因此项目采取以下控制措施：

1. 以题目原文、实物照片、模块丝印、数据手册和本地代码为依据，不凭空假设接口。
2. 所有电源、电平和电机方向先低风险验证，再接入自动控制。
3. 固件修改后进行代码检查和 Keil 编译验证，现场测试前保留已通过的 F/G/Y 回归模式。
4. 报告中对未完成或未实测功能使用“预留”“待联调”“需补充验证”等表述，避免把设计目标写成已通过结果。

## 9. 总结

本项目围绕题目基本 60 分功能建立了较完整的底盘控制方案：以 MSPM0G3507 为核心，使用非视觉灰度传感器进行黑线巡迹，结合编码器和 MPU6050 处理白区直行与关键点转换，并通过 ABCD 自动状态机完成巡迹联动。项目已完成主控、传感器、电机、编码器、巡线 PID、白区直行、IMU 校准、自动状态机和声光提示等关键底盘模块的开发和阶段性验证。

后续工作将集中在三方面：第一，继续提高 D 点入口和完整 ABCDA 联动的重复稳定性，并记录满足 30 s/40 s 要求的视频证据；第二，完成二维云台、执行光源或标记机构的独立供电和定点作业联调；第三，在底盘和执行机构稳定后尝试动态跟随、同步绘制和四圈连续运行等发挥项。通过分模块验证、保留回归测试模式和严格的供电安全策略，本设计具备继续完善为线上提交作品的工程基础。
